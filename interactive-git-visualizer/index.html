Simulated Commands

You can type these commands in the terminal console or use the action buttons:

git commit -m "message"
Creates a new commit on the current branch with a custom message.
git branch <name>
Creates a new branch at the current commit.
git checkout <branch-or-commit>
Switches your HEAD pointer to the specified branch or a specific commit hash (detached HEAD).
git merge <branch>
Merges the specified branch into your current branch (creates a merge commit or fast-forwards).
git rebase <branch>
Reapplies commits of current branch on top of another branch.
git reset --hard <commit-hash>
Moves current branch head back to the specified commit, discarding changes.

Git Area Lifecycle

  • Working Directory: Unsaved changes. Click "+ Modify/Create File" to add a file here.
  • Staging Area: Staged files. Click "git add" or run the command to stage them.
  • Local Repo: Run "git commit" to permanently store them into a commit node.
